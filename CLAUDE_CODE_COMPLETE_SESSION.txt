F2. MACHINE OPERATOR OFF-CUT RETURN FLOW
─────────────────────────────────────────────────────
Current: off-cut recorded at cutting confirmation,
no store admin verification step.

Add verification step:

After operator records off-cut in cutting confirmation:
  1. System sends alert to Store Admin:
     "Off-cut returning: [dimensions] from [machine]
     [operator name]. Please verify before accepting."

  2. Off-cut lot is created with status: Pending Verification
     (not Available yet)

  3. Operator physically returns off-cut to bay

  4. Store Admin opens Stock → Off-cuts Pending Verification
     Shows: lot number, expected dimensions, from machine
     Action: Verify and Accept | Reject (dimensions don't match)

  5. If Accepted: off-cut lot status → Available
     Inherits parent lot reservation automatically

  6. If Rejected: alert back to operator with reason
     Operator must re-measure and resubmit dimensions
