## Notes on resolving these

- **Issue #6 is the priority** — it's the only one where a working hard-gate control has a working, unguarded alternative path around it on the same data. Resolving it well (e.g. a single shared `canCompleteStage()` function that every completion path is forced to call) would also make Issues #7 and the broader permission fragmentation easier to reason about, since it collapses two systems into one.
- **Issues #1, #8, #9, #10** are all small, mechanical fixes once prioritized — they don't require architectural changes, just role-list corrections or a disabled-state addition.
- **Issues #4, #5, #7** are policy decisions as much as code changes — worth deciding what you actually want the behavior to be before writing the fix, since the "correct" answer depends on how your QC team actually operates, not just what the code currently does.

This file should be updated whenever an issue is resolved (move to a "Resolved" status with the date and what changed) or when new issues are found in future document-update sessions — not deleted, so there's a record of what's been checked and fixed over time.
